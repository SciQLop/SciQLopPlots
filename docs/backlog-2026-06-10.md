# Audit Backlog ‚Äî 2026-06-10

Findings from a six-agent parallel review (Model Contracts, Ownership & Lifecycle, Thread Safety, API & C++ Correctness, Error Handling, Performance & Quality) of **SciQLopPlots + NeoQCP** (387 files, 93k lines), run 2026-06-09. Items are prioritized **CRITICAL** (segfault / UB / data corruption / terminate), **HIGH** (race / leak / silently wrong result), **MEDIUM** (perf / API / silent UX). Items tagged `[NeoQCP]` are fixed in the NeoQCP repo.

Cross-corroboration note: C2/C3 (colormap UAF, invokeMethod TOCTOU) and H4/I5 (NaN interval, parallel binning) were each found independently by two agents.

Findings under **INVESTIGATE** are listed but skipped from the action plan until reproduced.

## CRITICAL

### ~~C1~~ ‚Äî FIXED 2026-06-10 (branch `fix/pipeline-lifetime`) ‚Äî `SimplePyCallablePipeline` ignores its `parent` argument ‚Üí leaks a live QThread per function-graph
- **File:** `src/DataProducer.cpp:235` (ctor); allocation at `src/SciQLopGraphInterface.cpp:100`
- **Bug:** The ctor takes `QObject* parent` but never forwards it to `QObject`. `SciQLopFunctionGraph` believes it parented the pipeline; its dtor is `= default`. The pipeline (and its `DataProviderWorker`) is never deleted, so `quit()/wait()` never runs ‚Äî the worker QThread runs forever and pins a Python callable reference. Every virtual-product graph created/destroyed in SciQLop accumulates an OS thread.
- **Fix:** Initialize `QObject(parent)` in the ctor (one line). Existing call sites then clean up via QObject parenting.
- **Confidence:** 90

### ~~C2~~ ‚Äî FIXED 2026-06-10 (NeoQCP branch `fix/async-pipeline-job-safety`, 74117d2: by-value capture + installResampleTransform re-bake; ASan reproducer) ‚Äî `[NeoQCP]` ColorMap2 transform captures `mGapThreshold` by reference ‚Üí worker-thread UAF
- **File:** `src/plottables/plottable-colormap2.cpp:17-72`; amplifier in `src/datasource/pipeline-scheduler.cpp` (dtor drains queue)
- **Bug:** The resample transform lambda captures `[&gapThreshold = mGapThreshold]` (a member). Pool jobs execute the transform *before* any destroyed-check; `removePlottable` deletes immediately and nothing joins in-flight jobs ‚Äî a heavy 2D resample reads the freed atomic. In `~QCustomPlot`, plottables die before the scheduler child and `waitForDone()` lets *queued* jobs start after the plottable is gone. Closing a plot during a spectrogram resample reliably touches freed memory. All other plottables capture by value/statelessly.
- **Fix:** Capture the gap threshold by value at job-bake time (re-bake on `setGapThreshold`, like `QCPHistogram2D::installTransform`). Also clear `mFastQueue`/`mHeavyQueue` at the top of `~QCPPipelineScheduler` before `waitForDone()`.
- **Confidence:** 85 (2 agents)

### ~~C3~~ ‚Äî FIXED 2026-06-10 (same NeoQCP commit 74117d2: mutex-guarded destroy flag held across check-and-invoke; scheduler dtor drops queued jobs ‚Äî covers the below-threshold drain note too) ‚Äî `[NeoQCP]` Async pipeline TOCTOU: `mDestroyed` check then `invokeMethod(self)` on a possibly-deleted pipeline
- **File:** `src/datasource/async-pipeline.h:172-182`, `async-pipeline.cpp:28-31`
- **Bug:** `~QCPAsyncPipelineBase` only stores `true` into a shared atomic. The worker job does `if (destroyed->load()) return; invokeMethod(self, ‚Ä¶)` ‚Äî the GUI thread can delete the pipeline between the load and the invoke. Permanently open UAF window affecting every pipeline plottable (Graph2, MultiGraph, ColorMap2, Histogram2D, Waterfall).
- **Fix:** Shared `QMutex + bool` held across check-and-invoke in the job, locked in the dtor; or `weak_ptr` delivery token; or post results through the scheduler (which *is* joined) and look the pipeline up under a lock.
- **Confidence:** 82 (2 agents)

### ~~C4~~ ‚Äî FIXED 2026-06-10 (NeoQCP branch `fix/waterfall-snapshot`, 6ab2e66, stacked on job-safety: immutable per-generation adapter via rebuildAdapter()) ‚Äî `[NeoQCP]` Waterfall adapter mutated by GUI thread while resampler jobs read it ‚Äî data race + inner-source UAF
- **File:** `src/plottables/plottable-waterfall.cpp:12-162`
- **Bug:** Waterfall hands the pipeline one long-lived *mutable* `QCPWaterfallDataAdapter` (all other sources are immutable snapshots). `draw()` reassigns `mOffsets`/`mNormFactors`/`mGain` on every paint; `setDataSource()` swaps the plain `shared_ptr mSource` ‚Äî while the pool job's L1 build reads those members via `keyAt()/valueAt()`. The job's `shared_ptr` keeps only the adapter alive, not the inner source: new data mid-job destroys the `QCPSoAMultiDataSource` the worker is iterating. Replot during a busy L1 build is the normal case on large waterfalls.
- **Fix:** Build a fresh immutable adapter per generation (own `shared_ptr` to inner source + value copies of offsets/norm/gain). Or keep offsets/gain out of L1 entirely and apply them at line-generation time (also fixes H6).
- **Confidence:** 85

### ~~C5~~ ‚Äî FIXED 2026-06-10 (branch `fix/pipeline-lifetime`) ‚Äî Pipeline-delivered `set_data` exceptions ‚Üí `std::terminate` (queued slot context)
- **File:** connection at `src/SciQLopGraphInterface.cpp:106-118`; throwers `src/SciQLopMultiGraphBase.cpp:129`, `src/SciQLopSingleLineGraph.cpp:81`, `src/SciQLopColorMap.cpp:81,105,112`, `src/SciQLopHistogram2D.cpp:97`, `src/SciQLopCurve.cpp:113`
- **Bug:** All `set_data` implementations throw on float32 x / shape mismatch. Function-graph data is delivered via queued signal from the `DataProviderWorker` thread; the throw escapes `QObject::event` ‚Üí terminate. A virtual-product callable returning float32 x (Speasy default) kills the app. `bindings.xml` auto-on only protects direct Python calls (see memory `qt_slot_exception_pitfall`).
- **Fix:** Validate-and-drop with `qWarning` at the pipeline boundary (in the `SciQLopFunctionGraph` lambda targets, or in `_collect_buffers` which already has the drop-batch + RuntimeWarning machinery).
- **Confidence:** 92

### ~~C6~~ ‚Äî FIXED 2026-06-10 (branch `fix/pipeline-lifetime`, typesystem-root auto-on) ‚Äî `exception-handling="auto-on"` missing for line/curve `set_data` ‚Äî direct Python calls can also terminate
- **File:** `SciQLopPlots/bindings/bindings.xml:488-501` (auto-on only on ColorMap/Histogram2D)
- **Bug:** `SciQLopMultiGraphBase`, `SciQLopSingleLineGraph`, `SciQLopCurve`, and the base interface throw with no translation ‚Äî `graph.set_data(x_f32, y)` unwinds a C++ exception through CPython frames. Per-function whack-a-mole already missed four classes.

- **Confidence:** 88

### ~~C7~~ ‚Äî FIXED 2026-06-10 (branch `fix/pipeline-lifetime`; `sqp::validation` wired into multigraph/single-line/curve set_data, resampler clamp, atomic `_line_cnt`) ‚Äî No x/y length validation in `set_data` ‚Üí out-of-bounds heap reads (3 paths)
- **Files:** `src/SciQLopMultiGraphBase.cpp:48-99` (`build_data_source`), `src/SciQLopSingleLineGraph.cpp:91-104`, `src/SciQLopCurveResampler.cpp:26-66` + `src/SciQLopCurve.cpp:98-122`
- **Bug:** y spans are constructed with `n = x.flat_size()`; `y.flat_size()` is never checked. `set_data(x_1000, y_500)` reads 500 elements past the y buffer at render. NeoQCP's `Q_ASSERT(size(col)==size(mKeys))` is tautological (span forced to x's length). Curve path reads `line_count() * count` y values; `_line_cnt` is additionally a plain `size_t` written from GUI thread, read from worker (data race). `sqp::validation::validate_xy/validate_xyz` (`Validation.hpp`, `MatchedBuffers.hpp`) already exist with **zero call sites**.
- **Fix:** Wire the existing validators into the five `set_data` entry points; clamp the curve resampler's line loop to `y.flat_size()/count`; make `_line_cnt` atomic or fold under the data mutex. Also covers `SciQLopWaterfallGraph::raw_value_at` (indexes y by x-derived idx) and `SciQLopNDProjectionCurves::set_data` pairs (lengths never cross-checked).
- **Confidence:** 85‚Äì88

### ~~H1~~ ‚Äî FIXED 2026-06-10 (branch `fix/audit-2026-06-10`: announced remove+replace in `_insert_node`, `removeIf` stripped from `add_child`, new `take_child`) ‚Äî ProductsModel: silent same-name child replacement corrupts row accounting
- **File:** `src/ProductsNode.cpp:64-70` (`add_child` removeIf), `src/ProductsModel.cpp:49-55` (`_insert_node`)
- **Bug:** `add_child()` does `removeIf(same name)` ‚Äî a structural removal with no `beginRemoveRows` ‚Äî while `_insert_node` already announced an insert at row N. Re-publishing a product (ordinary event, `add_node` is Python-exposed) leaves views/proxies believing N+1 rows exist; the recursive tree filter maps a row that resolves to nullptr; evicted nodes accumulate as orphaned QObjects. ProductsModel has zero `beginRemoveRows` anywhere.
- **Fix:** Look up the existing child first; update in place (emit `dataChanged`) or remove through a proper begin/endRemoveRows pair. Strip the `removeIf` from `add_child` so node containers never mutate structure invisibly.
- **Confidence:** 88

### H4 ‚Äî `[NeoQCP]` Adaptive resampling: wrong "last point of interval" when NaNs interspersed
- **File:** `src/datasource/algorithms.h:256-298` (`optimizedLineData` / `flushInterval`)
- **Bug:** `prev = intervalFirst + intervalCount - 1` assumes contiguous indices, but the scan skips NaNs without counting them ‚Äî `prev` can land on a NaN, appending a spurious line-break point. Upstream QCustomPlot uses the actual last iterator. Phantom gaps in dense gappy data.
- **Fix:** Track the index of the last accepted (non-NaN) point where min/max are updated; pass that to `flushInterval`.
- **Confidence:** 85 (2 agents)

### H5 ‚Äî `block_stats_column`: NaN first row poisons min/max forever
- **File:** `include/SciQLopPlots/DSP/Stats.hpp:57-58`
- **Bug:** `lo`/`hi` seeded unconditionally from row 0; comparisons against a NaN seed never succeed, so min/max stay NaN even with valid data following (mean/variance correct). Instrument data routinely starts with fill values. Python-reachable via the DSP `block_stats` binding.
- **Fix:** Seed from the first non-NaN element, or seed ¬±inf and convert untouched sentinels to NaN (matching count==0).
- **Confidence:** 85

### ~~H6~~ ‚Äî FIXED 2026-06-10 (same NeoQCP commit 6ab2e66: every parameter change rebuilds the adapter through setDataSource, invalidating line/L1/L2 caches) ‚Äî `[NeoQCP]` Waterfall gain/offset/normalize changes invalidate no cache ‚Äî replot draws stale data

- **Confidence:** 85

### ~~H8~~ ‚Äî FIXED 2026-06-10 (NeoQCP branch `fix/datasource-shape-validation`, ccc5f4f: warn + degrade-to-empty in all four ctors, QTest reproducers) ‚Äî `[NeoQCP]` Datasources: `Q_ASSERT` is the only shape validation ‚Äî UB in release, abort in debug
